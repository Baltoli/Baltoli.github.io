Detailed Description

Represents the result of a parse as a tree structure.

This structure owns all of the child nodes it contains.

Member Data Documentation

◆ children

struct parse_st* parse_st::children

The child nodes for this node.

This field is only used if the node is a non-terminal. In this case, the child nodes are those generated by the production rule for this non-terminal.

◆ length

size_t parse_st::length

The length of this parse result.

This field is not valid if this node is a non-terminal whose children have not yet been added. Once all children of this node are added, then the length can be computed and stored properly by calling parse_total_length.

◆ symbol

char* parse_st::symbol

The symbol this node represents.

If this node is a non-terminal, this is the non-terminal name. If it is a terminal, then the symbol is the literal string matched.

◆ terminal

bool parse_st::terminal

Marks whether this node is a terminal or a non-terminal.

This can't be reliably computed from other properties of the tree - it's perfectly valid for a non-terminal to be zero-length or have no children (if it's made up only of predicates, for example). It is better to set the property explicitly.

Set to false by default by parse_init.
